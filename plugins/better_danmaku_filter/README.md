# 智能弹幕精选（better_danmaku_filter）

NipaPlay 弹幕过滤插件，通过多维度评分算法智能筛选优质弹幕，过滤低质量内容。

在线演示：[www.retr0.xyz/better_danmaku_filter.html](https://www.retr0.xyz/better_danmaku_filter.html)

## 功能

- **质量评分**：根据弹幕长度、字符多样性、重复字符比例等维度打分
- **刷屏过滤**：检测短时间内重复发送的相同弹幕
- **相似去重**：基于编辑距离过滤近似重复弹幕
- **短弹幕过滤**：过滤过短的无意义弹幕
- **无意义短串过滤**：过滤无意义的短数字/字母串（如 `111`、`aaa`、`123`、`4567`），同时保留 `666`、`2333`、`awsl`、`hhh` 等常见网络用语
- **Emoji 处理**：纯 Emoji 弹幕可选择降权或直接过滤
- **英文/数字惩罚**：全英文/数字弹幕降低评分（纯数字重罚、网络用语轻微降权）
- **中文加分 / 优质词加分**：含中文的弹幕信息量更高给予加分，命中「前方高能」「细节」「名场面」等优质关键词额外加分
- **时间分布优化**：对稀疏时间段的弹幕适当加分，避免某段时间弹幕全部被过滤
- **类型区分过滤**：滚动弹幕与顶部/底部弹幕采用不同的去重和刷屏阈值

## 安装

将 `better_danmaku_filter.js` 导入 NipaPlay 插件系统即可。导入后在插件设置中启用。

## 配置

启用后点击插件旁的扳手图标进入设置，可调整以下参数：

### 数值参数

| 参数 | 说明 | 默认值 | 范围 |
|------|------|--------|------|
| 期望弹幕数 | 目标保留弹幕数（以千为单位），0 表示使用保留比例 | 0 | 0+ |
| 最终保留比例 | 保留弹幕的百分比 | 30% | 5-80 |
| 时间窗口 | 刷屏/去重检测的时间窗口（秒） | 5 | 1-30 |
| 最短弹幕长度 | 弹幕最小字符数 | 3 | 1-10 |
| 重复字符阈值 | 单字符占比超过此值视为重复 | 60% | 30-100 |
| 全英数惩罚 | 全英文/数字弹幕的扣分值 | 30 | 0-80 |

### 开关

| 开关 | 说明 | 默认 |
|------|------|------|
| 过滤相似弹幕 | 滚动弹幕相似度 > 93%、顶部/底部 > 82% 时去重 | 开 |
| 过滤刷屏弹幕 | 滚动弹幕同一内容 3 次以上、其他类型 2 次以上标记为刷屏 | 开 |
| 过滤过短弹幕 | 长度小于「最短弹幕长度」的弹幕直接过滤 | 开 |
| 过滤无意义短串 | 过滤 `111`/`aaa`/`123` 等无意义短数字字母串，保留常见网络用语 | 开 |
| 允许纯Emoji降权 | 开启时纯 Emoji 弹幕扣 10 分，关闭时扣 40 分 | 开 |
| 高级语义评分 | 弹幕稀疏时间段加分 + 时间分布优化 | 开 |

## 评分规则

每条弹幕基础分 50 分，根据以下维度调整：

- **长度**：2 字以下 -30，3-5 字 +5，6-15 字 +15，16-30 字 +10，30 字以上 +5
- **字符多样性**：字符种类占比越高加分越多（最高 +20）
- **重复字符**：单字符占比超过阈值时扣分
- **纯 Emoji**：降权 -10 或过滤 -40
- **中文加分**：含中文字符 +8
- **全英文/数字**：分级扣分——纯数字重罚（惩罚值 +15），普通全英数按配置扣分，命中网络用语白名单仅轻微降权（惩罚值 ×0.3）
- **标点过多**：标点占比 > 40% 时 -10
- **优质关键词**：命中「前方高能」「细节」「名场面」等 +8
- **弹幕类型**：所有类型平等竞争，无额外加分
- **稀疏时段**：所在时间段弹幕少于 3 条时 +15

最终按评分从高到低排序，取前 N 条保留。

## 工作流程

```
加载弹幕 → 短弹幕预过滤 → 无意义短串过滤 → 刷屏检测 → 相似去重 → 质量评分
→ 时间分布优化 → 按比例/期望数截取 → 输出精选结果
```

## 依赖

- NipaPlay >= 1.10.6
- 权限：`danmaku.modify`（修改弹幕显示）、`ui.dialog`（显示提示）

## 更新日志

### v1.1.3

- **修复**：期望弹幕数优先级高于最终保留比例；当「期望弹幕数」大于 0 时，按 `期望值 × 1000` 作为目标保留数，不再参与比例截取
- **修复**：若原始弹幕数不超过目标保留数（例如期望 4k、实际 2707 条），直接跳过精选过滤，避免被 `30%` 等最终保留比例误过滤
- **优化**：跳过过滤时显示提示，便于确认当前弹幕数未超过目标保留数

---

### v1.1.2

- **新增**：无意义短串过滤——过滤 `111`、`aaa`、`123`、`4567` 等无意义的短数字/字母串弹幕，同时通过白名单保留 `666`、`2333`、`awsl`、`hhh`、`yyds` 等常见网络用语
- **新增**：中文内容加分（+8），引导精选偏好信息量更高的弹幕
- **新增**：优质关键词加分（「前方高能」「细节」「名场面」「破防」「彩蛋」等 +8）
- **新增**：标点符号占比过高降权（> 40% 时 -10）
- **优化**：全英文/数字弹幕改为分级惩罚——纯数字重罚、普通全英数按设定值、网络用语仅轻微降权
- **优化**：补充 `hasCJK`、`isPureDigits`、`isSingleCharRepeat` 等检测函数

---

### v1.1.1

- **适配**：支持 NipaPlay 弹幕处理插件 Pipeline 链式处理，声明 `priority: 50`，在弹幕流量控制器等之前执行过滤
- **调整**：最低宿主版本提升至 1.10.6（需宿主支持 Pipeline 链式事件分发）

---

### v1.1.0

- **修复**：移除顶部/底部弹幕的 +10 分人工加成，所有弹幕类型凭内容质量平等竞争

- **修复**：去重过滤按类型区分阈值——滚动弹幕提升至 93%，大幅减少"哈哈哈"等高频相似内容被批量误杀的问题

- **修复**：刷屏过滤按类型区分——滚动弹幕同一内容允许 3 次（原来统一 2 次），降低误判率

- **优化**：整体算法对滚动弹幕更加友好，改善"几乎只剩顶部弹幕"的问题
